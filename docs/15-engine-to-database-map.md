## Ownership Conflicts Resolved

- `profiles` is not an authentication credential store; Supabase Auth remains authoritative.
- Progress owns completion; Certificate only consumes eligibility facts.
- Question Bank owns keys; Assessment receives only server-side evaluation access and learner-safe form projections.
- Payment owns provider transaction facts; Subscription owns entitlement state.
- Hiring Partner owns partner verification; Candidate Visibility alone owns PII reveal.
- Event, Workflow, Analytics and Audit each store different facts: transport, orchestration, measurement and accountability respectively.

## Phase 1 Additions Requiring ADR

The previous database blueprint intentionally deferred dedicated event outbox/checkpoint, workflow instance/step, recommendation result, security alert/case, support-access grant and gamification rule/badge tables. They must be specified and approved before their engines are implemented; this document does not authorize migrations.
