# Master Table Catalog

## How To Read This Contract

This catalog defines 169 physical application tables. `certificate_public_views` is documented separately as the one protected projection and is not included in the physical-table count.

### Column Bundles

- **`TG` mutable global:** `id uuid PK=generated`, `created_at timestamptz=now`, `updated_at timestamptz=now`, `created_by uuid? -> profiles`, `updated_by uuid? -> profiles`.
- **`TT` mutable tenant:** `TG` plus `organization_id uuid -> organizations`.
- **`IG` immutable global fact:** `id uuid PK=generated`, `occurred_at timestamptz=now`, `actor_profile_id uuid? -> profiles`; no `updated_at`/`deleted_at`.
- **`IT` immutable tenant fact:** `IG` plus `organization_id uuid -> organizations`.
- **`J` junction:** listed FK columns form composite PK plus `created_at timestamptz=now`; no surrogate ID unless row says otherwise.
- `?` means nullable. `=x` is the default. Text status fields use the logical domain in `docs/22-master-enum-catalog.md`.

### Scalar Type Dictionary

Compact bare fields are still fully typed by this contract:

- `name`, `title`, `key`, `slug`, `description`, `reason`, `reason_code`, `type`, `status`, `source`, `provider`, `model`, `locale`, `region`, `classification`, `content_hash`, `sha256`, `correlation_id`, `causation_id`, `idempotency_key` and fields ending `_hash`, `_code`, `_reference`, `_path`, `_url` or `_key` are `text` unless a logical enum/domain or explicit type is named.
- fields ending `_id` are `uuid`; a relationship is present only when `-> target` is shown. Polymorphic typed IDs are explicitly documented exceptions.
- `version`, `position`, `attempt`, `attempts`, `sequence_no`, token counts, duration/latency/percentage integers and fields ending `_count` are `integer` unless `bigint`/`numeric` is explicit.
- fields ending `_at` are `timestamptz`; fields ending `_on`/`_date` and named period dates are `date`; nullable/default markers still apply.
- `is_*`, `has_*`, `eligible`, `enabled`, `required`, `passed`, `active`, `signature_valid`, `human_review_required` and other fields with explicit true/false defaults are `boolean`.
- JSONB, arrays, `citext`, `char(n)`, `numeric`, `bigint`, `tsvector` and `vector` are always written explicitly.
- A bare field is non-null with no default. A row may override this dictionary explicitly; explicit type wins.

`deleted_at` appears only when explicitly listed. Immutable rows never contain it.

### Physical And Governance Codes

- **Partition:** `P0` none initially; `P1` monthly range on event time; `P2` monthly range with organization subpartition candidate; `P3` hash organization candidate only after benchmark.
- **Version:** `V0` no row version; `V1` optimistic integer `version=1`; `V2` immutable artifact version with owner/version uniqueness.
- **Retention:** `R1` account +30d; `R2` learning +3y; `R3` credential +7y; `R4` evidence 7y/legal hold; `R5` finance 8y; `R6` operations 90d/payload 30d; `R7` AI content 30d/usage 13mo; `R8` analytics raw 13mo/aggregate 3y; `R9` consent/grant lifecycle +R4; `R10` platform lifetime/version history.
- **Encryption:** `E0` no application-layer requirement; `E1` Supabase encryption at rest/TLS plus access controls; `E2` application encryption or one-way hash for named fields.
- **Audit:** `A0` ordinary domain telemetry; `A1` mutations append `audit_events`; `A2` table is append-only evidence/audit.
- **Realtime:** `RT0` disabled; `RT1` enabled for RLS-protected Postgres Changes; `RTB` server Broadcast preferred at scale.
- **Integrations:** `St` Supabase Storage, `Srch` search source/projection, `An` analytics source, `-` none.
- **Security:** `S0` public/reference, `S1` authenticated global, `S2` self, `S3` tenant member, `S4` scoped staff, `S5` tenant admin, `S6` platform operator, `S7` compliance/security, `S8` service-only, `S9` exact-token public projection. Full matrix is in `docs/28-database-security-matrix.md`.

Every row explicitly states generated columns (`Gen:none` where absent), partition/version/retention, governance and future expansion (`F:`).

## Protected Projection

`certificate_public_views` is the only anonymous projection approved by this contract. It is populated from `certificates` plus latest `certificate_status_events`, exposes no internal profile/organization IDs, requires an exact high-entropy verification token and is accessed through a rate-limited server function/Route Handler. Whether it is a normal view, materialized view or maintained table is Open Decision DB-011; its public field contract is fixed.
