# Student Submission Guide

Learners can access only assignment deliveries allocated to their identity in the active organization. Starting work creates a draft submission and version. Text and private-file metadata may change only while that version is a draft.

Submitting seals the version, records status evidence and creates a grading queue item. Resubmission creates a new version and never replaces earlier work. Files use the private assignment bucket, governed metadata, ownership checks, MIME and size validation, checksum metadata and quarantine state.
